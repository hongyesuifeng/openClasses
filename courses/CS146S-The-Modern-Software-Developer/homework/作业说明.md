## Week 1: Prompting Techniques（提示工程实践）

### 作业目标
通过实践掌握 6 种核心提示工程技术。

### 技术清单
1. **K-shot Prompting（少样本提示）**
   - 文件：`k_shot_prompting.py`
   - 任务：通过提供示例来改善模型输出

2. **Chain-of-Thought（思维链）**
   - 文件：`chain_of_thought.py`
   - 任务：引导模型展示推理过程

3. **Tool Calling（工具调用）**
   - 文件：`tool_calling.py`
   - 任务：让模型调用外部工具

4. **Self-Consistency Prompting（自洽性提示）**
   - 文件：`self_consistency_prompting.py`
   - 任务：通过多次采样提高答案可靠性

5. **RAG（检索增强生成）**
   - 文件：`rag.py`
   - 任务：结合外部知识库生成回答

6. **Reflexion（反思）**
   - 文件：`reflexion.py`
   - 任务：让模型反思并改进自己的输出

### 环境要求
- Python 3.12
- Ollama（本地运行 LLM）
  - 需要下载模型：`mistral-nemo:12b` 和 `llama3.1:8b`

### 提交要求
- 完成所有 6 个技术文件中的 `TODO` 部分
- 设计并优化提示词
- 通过测试脚本
- 保存最终的提示词和输出

### 评分标准（60 分）
- 每个技术 10 分

## Week 3: Build a Custom MCP Server（构建自定义 MCP Server）

### 作业目标
设计并实现一个 Model Context Protocol (MCP) 服务器，封装真实的外部 API。

### 学习目标
- 理解 MCP 核心能力：tools、resources、prompts
- 实现带类型参数的工具定义和错误处理
- 遵循日志和传输最佳实践
- 可选：实现认证流程

### 要求

#### 1. API 选择
选择一个外部 API 并记录要使用的端点：
- 天气 API
- GitHub Issues
- Notion Pages
- 电影/电视数据库
- 日历
- 任务管理器
- 金融/加密货币
- 旅游
- 体育统计
- 其他...

#### 2. 核心功能
- 至少暴露 2 个 MCP 工具
- 实现基本的韧性：
  - 优雅处理 HTTP 失败、超时、空结果
  - 尊重 API 速率限制

#### 3. 部署模式（二选一）

**本地部署（STDIO）**：
- STDIO 服务器
- 可在本地运行
- 可被 Claude Desktop 或 Cursor 等 AI IDE 发现

**远程部署（HTTP）**：
- HTTP 服务器
- 可通过网络访问
- 可被 MCP 客户端或 Agent 运行时调用
- ⭐ 额外加分

#### 4. 可选：认证（额外加分）
- 通过环境变量支持 API Key
- 或 OAuth2 风格的 Bearer Token

### 提交要求
- `week3/server/` 目录下的源代码
- `week3/README.md` 包含：
  - 前置条件、环境设置、运行说明
  - 如何配置 MCP 客户端
  - 工具参考：名称、参数、示例输入/输出

### 评分标准（90 分）
- **功能性（35 分）**：实现 2+ 工具，正确的 API 集成，有意义的输出
- **可靠性（20 分）**：输入验证、错误处理、日志、速率限制意识
- **开发体验（20 分）**：清晰的设置/文档、易于本地运行、合理的文件夹结构
- **代码质量（15 分）**：可读代码、描述性命名、最小复杂度、适当的类型提示
- **额外加分（10 分）**：
  - +5：远程 HTTP MCP 服务器
  - +5：正确实现认证

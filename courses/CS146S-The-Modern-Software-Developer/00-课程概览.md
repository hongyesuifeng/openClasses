## 课程简介

**CS146S: The Modern Software Developer（现代软件开发者）** 是斯坦福大学计算机系于 2025 年秋季开设的一门全新课程。这门课程由 Mihail Eric 主讲，他是斯坦福校友，曾在 Amazon Alexa 担任技术主管，创办过 YC 孵化的 AI 编程公司。

这是斯坦福历史上**第一门专注于 AI 软件开发**的课程，全面探索 AI 如何变革软件开发的方方面面。

## 课程核心理念

### 1. Human-Agent Engineering, Not Vibe Coding
课程开篇就提出了一个核心理念：**人机协作工程，而非氛围式编程**

纯粹靠"感觉"让 AI 生成代码（所谓 vibe coding），并不能产出生产级软件。真正的现代开发者，需要学会像管理一群"热情但稚嫩的 AI 实习生"一样，给它们提供：
- 清晰的上下文（Context）
- 明确的指令（Instructions）
- 合理的架构（Architecture）

### 2. LLMs Are Only as Good as You Are
另一个核心洞见：**LLM 的上限就是你的上限**

如果有人说"AI 在我的代码库上不好用"，通常意味着即便是一个人类新手进入这个代码库也会一头雾水。问题不在 AI，在于代码本身缺乏清晰的结构和上下文。

### 3. 角色转变
从"写代码的人"变成"**AI Agent 的管理者**"

### 4. 全面拥抱 AI
正如 Mihail Eric 在课堂上所说：
> "如果你能不写一行代码就完成整门课，那更好。"

## 核心知识点总结

### 1. Prompt Engineering（提示工程）
- 如何设计有效的提示词
- 上下文提供技巧
- 思维链（Chain-of-Thought）提示
- ReAct 框架应用

### 2. Agent Architecture（Agent 架构）
- 工具调用（Tool Calling）
- 函数调用（Function Calling）
- Model Context Protocol (MCP) - 模型上下文协议
- Coding Agent 设计模式

### 3. AI IDE（AI 集成开发环境）
- Cursor
- Claude Code
- 上下文管理策略
- PRD（产品需求文档）撰写

### 4. 现代开发工具链
- AI 增强终端（Warp）
- 安全扫描工具（Semgrep）
- 代码审查工具（Graphite）
- 部署平台（Vercel）

### 5. AI 测试与安全
- SAST（静态应用安全测试）
- DAST（动态应用安全测试）
- AI 生成测试用例
- 安全漏洞检测
- 建立对 AI 代码的信任机制

### 6. AI 辅助 UI 构建
- 一键生成 Web 应用
- 快速原型开发
- 设计普惠化
- 端到端应用构建

### 7. DevOps 与监控
- 部署后的 Agent
- 可观测性（Observability）
- 自动化事件响应
- AI 参与的运维流程

## 为什么这门课值得关注？

这门课回答了一个所有开发者都在问的问题：**AI 时代，我该如何工作？**

不是"AI 会不会取代我"——这是焦虑的问题。而是"如何与 AI 协作，让自己的生产力翻倍"——这是务实的问题。

CS146S 给出的答案很清晰：

1. **角色转变**: 从"写代码的人"变成"AI Agent 的管理者"
2. **能力要求**: 清晰的上下文表达、系统设计思维、质量把控能力
3. **工具链升级**: 全栈 AI 工具的实战掌握，从 IDE 到终端到安全审计
4. **信任与验证**: 建立对 AI 输出的信任机制，同时保持警惕

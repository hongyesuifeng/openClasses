AI编码的提问方式

每天都在用 AI 写代码，相同模型，问法不同，效果时好时坏，有啥区别？
═══════════════════════════════════════════════════════════
【三种问法对比】
任务：创建 Cocos Creator 装备面板

问法 A：简单直接
> 帮我创建一个 EquipmentMainView 面板。
结果：类名忘记加 $$ 后缀，配置文件经常漏掉

问法 B：给点参考
> 帮我创建面板，参考：
> export class RoleView$$ extends BaseView$$ { }
结果：☆ 记住了 $$ 后缀，但配置文件还是会漏

问法 C：给文档 + 给示例
> # 文档
> ## 配置注册：1.ViewEnum$$ 2.ViewCtrlEnum$$ ...
> ## 参考代码
> export class BagViewCtrl$$ extends BaseViewCtrl$$ {
>     protected viewClass: any = BagView$$;
> }
结果：✓ 6 个配置一步不落，方法名全都对
═══════════════════════════════════════════════════════════
【技术原理】

问法 A → Zero-Shot（零样本学习）
  不给任何参考，AI 用通用知识"猜"

问法 B → Few-Shot（少样本学习）
  给几个示例，AI 从中学习规律

问法 C → RAG + Few-Shot（检索增强+少样本）
  文档告诉 AI"有什么"，示例告诉 AI"怎么写"

示例能传递"看得见的模式"（如 $$ 后缀），但传递不了"完整的列表"（如 6 个配置文件）。文档 + 示例结合，才能既准确又对味。
═══════════════════════════════════════════════════════════
【什么时候用什么？】

• 通用问题（如"写个快速排序"）                 → 问法 A
• 统一代码风格                                          → 问法 B
• 项目特定任务（涉及框架方法、配置流程） → 问法 C
═══════════════════════════════════════════════════════════
【记住这个公式】

好结果 = 文档（有什么） + 示例（怎么写） + 清晰任务

以后别在对ai说一句话的需求，帮我写个xx系统了[不看]，你不想猜产品在想什么，AI也不想猜你在想什么！

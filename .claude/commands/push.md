执行完整的 git 工作流,将所有本地更改推送到远程仓库。

## 步骤

1. **检查状态** - 运行 `git status` 查看所有未跟踪和已修改的文件
2. **查看更改** - 运行 `git diff` 查看未暂存的更改,`git diff --staged` 查看已暂存的更改
3. **查看历史** - 运行 `git log -5 --oneline` 查看最近的提交历史,了解项目的提交信息风格
4. **分析更改** - 分析所有更改的内容,理解修改的性质和目的
5. **暂存文件** - 运行 `git add` 添加所有相关文件(排除敏感文件如 .env、credentials.json 等)
6. **创建提交** - 使用 `git commit` 创建提交,提交信息应:
   - 使用清晰简洁的中文描述
   - 遵循项目的提交信息风格
   - 包含 Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
   - 使用 heredoc 格式确保正确格式化
7. **验证提交** - 运行 `git status` 确认提交成功
8. **推送** - 运行 `git push` 将提交推送到远程仓库

## 提交信息格式

根据更改类型使用适当的前缀:
- `feat:` - 新功能
- `fix:` - 修复 bug
- `docs:` - 文档更改
- `refactor:` - 代码重构
- `chore:` - 构建/配置更改
- `test:` - 测试相关
- `style:` - 代码风格更改

提交信息应该:
- 清楚地说明"为什么"而不仅仅是"什么"
- 简洁但完整(通常 1-2 句话)
- 使用中文

## 注意事项

- 检查是否有不应该提交的敏感文件(.env、密钥、credentials 等)
- 如果有敏感文件,警告用户并排除它们
- 确保只提交有意义的更改,排除临时文件或调试代码

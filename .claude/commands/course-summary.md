description: 生成课程内容的结构化总结文档
---

根据课程资料生成完整的 11 章结构化课程总结文档。

## 参数

执行此命令前，请准备好以下信息：

- **周次** (WEEK_NUMBER): 课程周数，如 1, 2, 3...
- **课程主题** (COURSE_TOPIC): 本周课程主题的中文描述
- **源目录** (SOURCE_DIR): 课程资料所在目录（可选，默认为 courses/CS146S-The-Modern-Software-Developer/week-{周次}/）

## 执行步骤

1. **收集信息** - 确认用户提供的周次、课程主题和源目录

2. **读取课程资料** - 读取源目录中的所有课程文档，包括：
   - 讲义/幻灯片
   - 作业说明
   - 补充材料
   - 参考文档

3. **分析课程内容** - 提取课程的核心概念、技术原理和实战应用

4. **生成总结文档** - 按照以下 11 章标准格式生成总结：

   ```markdown
   # {课程主题完全指南}

   ## 📚 目录
   1.  [{主题}概述](#1-{主题}-概述)
   2.  [核心概念](#2-核心概念)
   3.  [技术原理](#3-技术原理)
   4.  [实现模式](#4-实现模式)
   5.  [实战应用](#5-实战应用)
   6.  [最佳实践](#6-最佳实践)
   7.  [进阶技巧](#7-进阶技巧)
   8.  [工具与生态](#8-工具与生态)
   9.  [实战案例深度解析](#9-实战案例深度解析)
   10. [核心思想总结](#10-核心思想总结)
   11. [参考资料](#11-参考资料)
   ```

5. **写入文档** - 将生成的总结文档保存到：
   - 路径: `/courses/CS146S-The-Modern-Software-Developer/week-{WEEK_NUMBER:02d}/{FILENAME}.md`
   - 命名规范: 使用大写字母和下划线，如 `PROMPT_ENGINEERING_COMPREHENSIVE_GUIDE.md`

## 输出规范

### Markdown 格式
- 使用 `## ` 标记主章节
- 使用 `### ` 标记子章节
- 使用 `---` 分隔章节
- 使用 emoji 作为章节和列表标记

### 代码块
- 使用三个反引号包裹代码
- 标注语言类型（python, javascript, yaml 等）
- 代码应简洁可运行

### 表格
- 使用 Markdown 表格格式
- 包含表头和对齐标记
- 对比表格至少 3 列：特性/说明/示例

### 内容质量
- **准确性**: 确保技术描述准确
- **完整性**: 覆盖所有核心内容
- **可读性**: 语言简洁明了
- **实用性**: 代码示例可运行
- **结构化**: 层次清晰，便于导航

## 注意事项

- 保持与已有总结文档（如 week-01、week-02）的格式一致性
- 技术术语保留英文，首次出现时附中文解释
- 文档大小控制在 800-1200 行
- 参考资料应包含可点击的链接
